# v3.1.8 Dev-Tooling Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `interrogate` with `docstr-coverage` (resolving PYSEC-2022-42969 by dropping the archived `py` 1.11.0 transitive), wire `pip-audit` into CI via a new GitHub Actions workflow + `make audit` target, surface `bayesian-sindy-workflow` in two agent delegation tables, and bump all plugins to v3.1.8.

**Architecture:** Four-commit sequence A→B→C→D mirroring the v3.1.5/v3.1.6/v3.1.7 cadence. A enables B (clean lockfile means B's bare `pip-audit` runs green with no `--ignore-vuln`). C and D are independent. Per-commit validator gate is the same 8 dimensions as v3.1.7 but the `pip-audit` dimension flips from `1 ignored` to `No known vulnerabilities found` from Commit A onward. No SKILL.md edits → no skill-budget gate, no xref delta, no critical trim fallbacks.

**Tech Stack:** Python 3.13+, `uv`, `docstr-coverage`, `pip-audit`, GitHub Actions (`astral-sh/setup-uv@v5`), Markdown (agent delegation tables), JSON (plugin manifests).

**Spec:** [`docs/superpowers/specs/2026-04-11-v3.1.8-dev-tooling-polish-design.md`](../specs/2026-04-11-v3.1.8-dev-tooling-polish-design.md)

**Prior context:** v3.1.7 landed 5 commits (`f4f40a70`, `d6f50d47`, `b057f70f`, `91a45834`, `e595fbf1`) with `pip-audit` shipped as a dev dependency and an inline `--ignore-vuln PYSEC-2022-42969` escape valve for the upstream blocker. v3.1.8 retires that escape valve.

**Spec correction discovered during plan authoring:** The spec said `interrogate` lives under `[dependency-groups].dev`. **It actually lives under `[dependency-groups].docs`** (`pyproject.toml` line 43). The plan uses the correct group. Spec also asked "v4? v5?" for `setup-uv` — the existing `.github/workflows/docs.yml` line 30 confirms **v5**.
