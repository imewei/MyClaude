### pip-audit gate

After Commit A lands, the per-commit validator gate also runs `pip-audit`:

```bash
uv run pip-audit 2>&1 | tail -10
```

**Pass criterion**: no HIGH or CRITICAL findings. LOW / MEDIUM acceptable but noted in commit messages.

**If pip-audit reports HIGH/CRITICAL**:
- Direct dev dep: bump in `pyproject.toml`, `uv sync`, re-run
- Transitive: bump the parent direct dep to pull in a patched transitive
- **No fix available (genuine upstream blocker)**: use inline `pip-audit --ignore-vuln <CVE-ID>` in the validator-gate command with a justification comment in the commit message. This is an escape valve for genuine upstream blockers only — a CVE whose vulnerable code path is verifiably never exercised in this repo, where the dep-of-the-dep is unmaintained/archived, and where no fix exists in any release.
- **Scope distinction**: inline `--ignore-vuln` flags in the gate command are IN scope for v3.1.7 (they are zero infrastructure). A persistent `pip-audit-ignore` config file with structured ignore rules is OUT of scope for v3.1.7 (v3.1.8+ scope — see Non-goals below). Use the inline flag form when needed in v3.1.7; if the list of ignores grows beyond 1-2 entries, escalate to v3.1.8 where the config-file infrastructure is planned.

Commit A's own gate will surface any pre-existing CVE before any content work starts. Any CVE accepted under the escape-valve rule must have its CVE ID, severity, upstream-fix status, and in-repo code-path analysis recorded verbatim in the Commit A commit message.

**Worked example — Lorenz-63 — Stages 1-5 breakdown**

**Stage 1: Generate ground-truth data** (~150 tokens, ~25 lines)
Lorenz-63 via `scipy.integrate.solve_ivp` with fixed random seed, add Gaussian noise, compute central-difference derivatives.

```python
import numpy as np
from scipy.integrate import solve_ivp

def lorenz(t, u, σ=10.0, ρ=28.0, β=8/3):
    x, y, z = u
    return [σ*(y - x), x*(ρ - z) - y, x*y - β*z]

t_eval = np.linspace(0, 10, 5000)
sol = solve_ivp(lorenz, (0, 10), [1.0, 1.0, 1.0], t_eval=t_eval, rtol=1e-10)
X = sol.y.T                                          # (5000, 3)

rng = np.random.default_rng(0)
X_noisy = X + 0.01 * X.std(axis=0) * rng.standard_normal(X.shape)
dXdt = (X_noisy[2:] - X_noisy[:-2]) / (t_eval[2:] - t_eval[:-2])[:, None]
X_mid = X_noisy[1:-1]                                # (4998, 3)
```

**Stage 2: Build candidate library** (~120 tokens, ~15 lines)
Second-order polynomial library: 1, x, y, z, x², y², z², xy, xz, yz (10 features). Known ground truth: 7 active terms across 3 state equations.

```python
def build_library(X):
    x, y, z = X.T
    return np.column_stack([
        np.ones_like(x), x, y, z,
        x*x, y*y, z*z, x*y, x*z, y*z,
    ])

Theta = build_library(X_mid)                         # (4998, 10)
# Ground truth: dx/dt = -10x + 10y, dy/dt = 28x - y - xz, dz/dt = -8/3 z + xy
```

**Stage 3: Fit horseshoe prior with NumPyro + NUTS** (~200 tokens, ~30 lines)
```python
import jax, jax.numpy as jnp, numpyro, numpyro.distributions as dist
from numpyro.infer import MCMC, NUTS

def bayesian_sindy(Theta, dXdt):
    n_feat, n_states = Theta.shape[1], dXdt.shape[1]
    tau = numpyro.sample("tau", dist.HalfCauchy(0.1))
    lam = numpyro.sample("lam", dist.HalfCauchy(jnp.ones((n_feat, n_states))))
    Xi  = numpyro.sample("Xi",  dist.Normal(0.0, tau * lam))
    sigma = numpyro.sample("sigma", dist.HalfCauchy(jnp.ones(n_states)))
    numpyro.sample("obs", dist.Normal(Theta @ Xi, sigma), obs=dXdt)

kernel = NUTS(bayesian_sindy, target_accept_prob=0.9)
mcmc = MCMC(kernel, num_warmup=1000, num_samples=2000, num_chains=4, progress_bar=False)
mcmc.run(jax.random.PRNGKey(0), Theta=jnp.asarray(Theta), dXdt=jnp.asarray(dXdt))
samples = mcmc.get_samples()                         # {"Xi": (8000, 10, 3), ...}
```

**Stage 4: Diagnostics + PSIS-LOO** (~180 tokens, ~25 lines)
```python
import arviz as az
idata = az.from_numpyro(mcmc)

print(az.rhat(idata))                                # all < 1.01
print(az.ess(idata, method="bulk"))                  # > 400 per parameter

loo_full = az.loo(idata, pointwise=True)
print(loo_full)
```
Plus cross-link to `mcmc-diagnostics` for the full convergence workflow.

**Stage 5: Extract inclusion probabilities and credible intervals** (~150 tokens, ~20 lines)
```python
Xi_samples = samples["Xi"]                            # (8000, 10, 3)
inclusion_prob = (jnp.abs(Xi_samples) > 0.1).mean(axis=0)   # (10, 3)
terms = ["1", "x", "y", "z", "x²", "y²", "z²", "xy", "xz", "yz"]
for s, state in enumerate(["dx/dt", "dy/dt", "dz/dt"]):
    print(f"\n{state}:")
    for i, t in enumerate(terms):
        p_in = inclusion_prob[i, s]
        if p_in > 0.9:
            q05, q95 = jnp.quantile(Xi_samples[:, i, s], jnp.array([0.05, 0.95]))
            print(f"  {t:4s}  P(in)={p_in:.2f}  [{q05:+.3f}, {q95:+.3f}]")
```
Expected: flags the 7 ground-truth-active terms with `P(in) > 0.9` and shows credible intervals that straddle the known σ=10, ρ=28, β=8/3.
