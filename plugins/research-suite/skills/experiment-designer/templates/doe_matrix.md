# Design-of-experiments matrix

The DoE is the concrete list of runs (or simulations) that the measurement will execute, derived from the factors the study varies and the response being measured.

## Selection guide

Four common patterns. Pick based on the question type and the factor count.

### Full factorial

All combinations of factor levels. Use when:
- Small factor count (up to 5 factors, each at 2-3 levels)
- All interaction terms are of scientific interest
- Run cost is low (simulations, cheap measurements)

Run count: $\prod_i \ell_i$ where $\ell_i$ is the number of levels for factor i.

### Fractional factorial

Subset of full factorial, carefully chosen to resolve main effects and low-order interactions while dropping high-order ones. Use when:
- Factor count is higher (5-15 factors, mostly at 2 levels)
- High-order interactions are expected to be negligible
- Run cost is moderate

Run count: $2^{k-p}$ where p is the fractional reduction. A resolution-IV design separates main effects from two-factor interactions but aliases some two-factor interactions with each other.

### Response surface

Designed to fit a quadratic model in the factors. Use when:
- Nonlinear response is expected
- The goal is to identify an optimum or characterize curvature
- Factor count is small (2-5)

Typical: central composite or Box-Behnken designs.

### Latin hypercube

Each factor is sampled N times, with each sample at a distinct level, and the combinations distributed to cover the space uniformly. Use when:
- Continuous factors
- Exploration rather than hypothesis testing
- Large factor count where full or fractional factorial is infeasible

## Template for the matrix

List the factors, the levels, the design choice, the rationale, and the full run table.

```
## Factors and levels

| Factor | Units | Type | Levels |
|--------|-------|------|--------|
| Volume fraction (phi) | dimensionless | continuous | 0.45, 0.50, 0.55 |
| Shear rate (gamma_dot) | 1/s | continuous | 0.1, 1.0, 10.0 |
| Temperature | K | continuous | 293, 298, 303 |
| Batch | identifier | categorical | A, B, C (fixed; 3 independent preparations) |

## Design choice

Full factorial 3 × 3 × 3 × 3 = 81 runs. Justified: run cost low (simulation + post-hoc XPCS extraction per run), interactions between phi and gamma_dot are of primary interest, and the batch factor requires full replication to separate batch effects from systematic variation.

## Run order

Randomized within blocks of batch to avoid systematic drift. See attached CSV generated by scripts/doe_generator.py.

## Blocking and replication

- Blocking: by batch (one batch per day)
- Replication: within-batch, each (phi, gamma_dot, T) combination measured in triplicate
- Total measurements: 81 design points × 3 replicates = 243
```

## Checks after generation

Before accepting the design:
- Run the generator and verify the run count matches expectation
- Verify factor balance (each level appears the same number of times)
- Confirm randomization (run order not correlated with any factor)
- Confirm feasibility (are 243 measurements actually achievable in the available time?)

If any check fails, either the design is wrong or the plan is over-ambitious.
